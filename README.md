# 🗣️ speaktrace

Meeting annotation and speaker-attribution tooling for high-fidelity conversation intelligence.

## Why this exists

This repo is the public, reusable core of a larger production workflow. It is designed to be clean, modular, and easy to adopt in other projects.

## Current scope

- Multi-speaker segmentation + attribution\n- Meeting timeline and event annotation\n- Speaker-aware summarization scaffolds\n- Export-friendly structured outputs

## Near-term roadmap

1. Release speaker-attribution baseline API\n2. Add meeting event taxonomy + annotation schema\n3. Add quality metrics for diarization/attribution\n4. Publish reproducible sample notebook + CLI

## Related work

- Private implementation roots: caf-interview-analysis, mining-meetings, caf_meeting_podcast.

## Status

Active build-out. Initial public baseline is focused on clean APIs, examples, and strong docs.
